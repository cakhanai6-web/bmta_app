# **브타 앱 화면별 설명서**

### 1. 광장 피드
현재 내 위치 기반해서 내 위치와 관련된 브타 피드들만 추려서 노출
브타 피드 글 등록 시 현재 위치 기반으로 등록 시점에 어느 지역인지 DB에 남으며, 휴대폰 이용 시 브타 앱은 위치 정보를 계속 불러온다.
그래서 앱을 열고 광장 피드 메뉴로 진입하면 현재 나의 위치(군/구) 기준으로 군/구에 등록된 피드만 리스트업하여 노출
광장 피드의 디자인, UI/UX 및 정책은 브타 피드와 동일
광장 피드 페이지 상위에는 현재 브타 앱에서 인식하고 있는 나의 위치 정보 노출되고 그 오른쪽에는 위치 새로고침 아이콘 노출
새로고침 아이콘을 노출하면 내 위치 정보를 다시 불러온다. 만약에 군/구가 변경됐다면 피드도 새로 불러온다.
그리고 현재 위치 정보와 새로고침 아래 영역은 브타 피드와 같이 카테고리 필터와 검색 영역 그리고 핫이슈와 브타 피드들이 리스트업되어 노출된다.

### 3. 브타 피드
**Blind(블라인드) 앱의 구조와 방식을 거의 그대로 채택**

앱을 실행하면 최초로 진입하는 화면
페이지 구성은 최상단에 대화 주제 토픽 필터가 보여지고 좌우 스와이프를 통해 확인 가능하다.
대화 주제 토픽은 전체/실시간톡/사람을 찾아요/이용 후기 총 4개가 있고 각각 아이콘과 버튼명이 카테고리 필터로 보여진다.
원하는 주제만 보고싶으면 필터를 클릭하면되고 클릭 시 해당 주제로 피드들이 새로고침된다.

토픽 아래에는 검색창이 있고 현재 보고 있는 적용되어 있는 토픽+'제목+내용'을 기준으로 입력한 키워드가 일치하는게 있는지 검색한다.
검색 영역은 상단에 고정되고 스크롤을 내리면 검색 결과 피드들이 계속 보여진다.

피드 페이지 우측 하단에는 플로팅 버튼으로 글쓰기 버튼이 있고 클릭하면 글쓰기 페이지로 이동한다.
글쓰기 버튼 클릭 시 로그인 여부 판단하고 회원은 글쓰기 페이지로 이동, 비회원은 "회원만 이용할 수 있는 서비스입니다. 로그인 페이지로 이동할게요." Alert 창이 보여진다. [확인] 버튼을 클릭하면 로그인 페이지로 이동되고 로그인하면 글쓰기 페이지로 이동한다.

검색 아래 영역은 실시간 핫이슈 피드로 좋아요+조회수+댓글의 총합이 핫이슈 기준이 되면 사용자들에게 강조되어 보여진다.

제목과 상세 내용의 일부만 보여지고 클릭하면 상세페이지로 이동한다.
핫이슈 피드와 실시간 브타 피드 모두 브타 피드 페이지에서는 제목은 최대 두 줄까지 노출되고 상세 내용의 첫 번째 줄부터 3줄까지 보여진다.
그 이상인 경우 제목과 상세 내용 모두 말줄임표가 적용된다.
그리고 작성자의 프로필과 아이디 그리고 토픽, 피드 등록된지 몇 초전 혹은 몇 분전, 몇 시간전과 같이 보여진다.
24시간 이후에는 등록일로 YYYY-MM-DD 형식으로 보여진다.
피드 영역 좌측 하단은 좋아요 아이콘과 개수 - 댓글 아이콘과 등록된 개수 카운트값이 보여진다. 우측 하단에는 조회수가 보여진다.

브타 피드 클릭 시 로그인 여부를 체크하여 로그인된 회원은 상세페이지로 이동하고 로그인이 안된 회원은 "회원만 이용할 수 있는 서비스입니다. 로그인 페이지로 이동할게요." Alert 창이 보여진다. [확인] 버튼을 클릭하면 로그인 페이지로 이동되고 로그인하면 클릭했던 피드의 상세페이지로 이동한다.

상세페이지에서는 전제 제목과 전체 본문 내용을 확인할 수 있다.
제목 위 영역은 좌측에 [< 브타] 버튼이 있고 우측에는 알림 아이콘과 즐겨찾기 아이콘 그리고 더보기(...) 버튼이 있다.
[< 브타] 버튼을 클릭하면 피드 목록페이지로 이동한다.
알림 아이콘을 클릭하면 해당 피드에 새로운 댓글이 달린 경우 내 폰으로 PUSH 알림이 발송된다.
즐겨찾기 아이콘을 클릭하면 내 정보 메뉴 > 즐겨찾기 메뉴에서 내가 등록한 피드들을 볼 수 있는데 여기에 추가된다.

즐겨찾기를 등록할 수 있는 개수는 최대 30개까지로 제한하고 30개까지 즐겨찾기를 등록해놨는데 더 추가하면 "즐겨찾기 최대 개수는 30개 입니다." 라는 Alert이 보여진다.

더보기 버튼을 클릭하면 레이어팝업으로 [차단하기] 버튼과 [신고하기] 버튼 [차단 및 신고하기] 버튼이 보여진다.
[차단하기] 버튼을 클릭하면 해당 사용자의 피드 및 댓글의 내용은 더이상 보여지지 않고 "차단된 사용자입니다." 라는 문구만 보여진다.

[신고하기] 버튼을 클릭하면 차단하진 않고 해당 게시글 및 댓글이 신고처리가 된다.
댓글 및 게시글의 신고 누적 카운트가 '10'이 되면 게시글과 댓글은 삭제되고 "신고된 게시글(댓글)입니다." 라고 보여진다.

[차단 및 신고하기] 버튼을 클릭하면 위의 두 가지가 모두 적용된다.

그리고 게시글의 제목과 전체 본문 내용 바로 아래에는 BM UP(좋아요) 버튼이 있다.

BMUP(좋아요)은 피드당 한 번 만 클릭 가능하다.
클릭하면 좋아요 아이콘이 움직이는 애니메이션 효과와 함께 버튼이 명암이 바뀌고 사용자에게 내가 BMUP을 눌렀고 제대로 반영됐구나가 보여진다.
이전에 BMUP(좋아요)를 클릭했던 피드라면 좋아요가 등록된 상태로 보여지고 한 번 더 누르면 BMUP(좋아요)가 취소된다.

아래 영역에는 댓글들이 보여지고 댓글들은 처음 등록된 것부터 마지막에 등록된 것까지 등록일 기준 오름차순으로 보여진다.
가장 높은 레이어에는 본인 댓글을 입력하는 필드가 있고 댓글 입력과 등록 버튼이 있다.
이미지 등 첨부 파일은 등록할 수 없고 오직 200자 제한의 텍스트만 입력할 수 있다.
**글 작성 시 제목이나 본문 작성 그리고 댓글 작성 시 마크다운이나 스크립트를 입력할 수 없다.**

댓글을 입력하면 등록한 계정의 닉네임과 몇 초전 혹은 몇 분전 등 등록한 시간대가 보여지고 아래 영역에는 댓글 내용이 보여진다.
그리고 그 아래에는 댓글달기라는 버튼이 있고 클릭하면 내가 클릭한 댓글에 대댓글을 달 수 있다.
남들이 작성한 댓글은 수정 삭제없고 댓글달기만 있다.
본인이 작성한 댓글은 댓글달기뿐만아니라 수정과 삭제 아이콘이 있다.
수정 아이콘을 클릭하면 입력된 댓글이 활성화되어 편집모드로 전환되고, 저장 버튼과 취소 버튼이 보여진다.
입력한 댓글을 편집모드 상태에서 수정하고 저장 버튼을 누르면 수정된 댓글로 저장되는거고 취소 버튼을 클릭하면 이전 편집 전 비활성화 모드로 돌아간다.
삭제 아이콘을 클릭하면 Confirm 창으로 "삭제하시겠습니까?" 가 노출된다. [확인] 버튼을 클릭하면 Confirm창이 사라지고 등록한 댓글은 삭제된다. [취소] 버튼을 클릭하면 Confirm창만 사라지고 바닥페이지로 돌아간다.

삭제한 댓글에 대댓글이 이전에 등록되어 있다면 "댓글이 삭제되었습니다." 라고 문구가 보여지고 등록되었던 대댓글은 계속 남아있다.
하지만 대댓글이 없는 댓글이면 댓글 리스트에서 안보인다.

댓글에 달린 대댓글은 디폴트로 2개까지 보여지고 3개 이상 달린 경우는 숨겨진 상태에서 "댓글 더보기 >" 버튼이 보여진다.
댓글 더보기 버튼을 클릭하면 5개씩 펼쳐지고 더이상의 대댓글이 없는 경우 버튼도 안보인다.

뒤로가기 제스처를 하거나 좌측 상단의 "< 브타" 버튼을 클릭하면 피드 목록 페이지로 이동한다.


### 4. 내정보
로그인을 안한 사용자는 하단 네비게이션 바의 내정보를 클릭하면 로그인 페이지로 이동된다.

로그인한 사용자인 경우 페이지에 가장 상단에는 동그란 프로필 이미지와 닉네임이 보여져
바로 아래에는 현재 나의 포인트가 몇인지 볼 수 있어
그리고 쪽지함이라는 메뉴와 설정, 공지사항, 고객센터 그리고 가장 마지막에는 로그아웃 버튼과 버전 정보가 있다.

쪽지함은 메뉴 클릭하면 보낸 쪽지함과 받은 쪽지함 전부를 확인할 수 있는 화면과 보낸 쪽지함, 받은 쪽지함을 필터로 구분할 수 있도록 해줘
쪽지함 메뉴에는 제목만 최대 한 줄로 보여지고 누가 언제 몇 시에 보냈는지 알 수 있어
클릭하면 상세 페이지로 이동하여 내용을 볼 수 있지 또 상세 페이지에서는 답장보내기 및 사용자 프로필을 선택해서 차단 혹은 신고 기능도 들어가면 좋겠다

**(추가)** 쪽지함 아래에는 즐겨찾기 메뉴가 있다.
메뉴를 클릭하면 본인이 브타 피드 메뉴에서 즐겨찾기를 등록한 것들이 보여진다.
즐겨찾기 메뉴에 저장된 피드 리스트는 간단한 게시판 형식으로 제목과 최대 2줄의 본문 내용이 보여진다.

그리고 각 리스트마다 우측에는 즐겨찾기 아이콘과 피드 등록일이 보여진다.
클릭하면 등록한 피드의 상세페이지로 이동한다.

만약에 즐겨찾기한 피드의 글쓴이가 삭제했을 경우 리스트에 "삭제된 피드입니다." 라는 문구 노출

즐겨찾기 아이콘을 재클릭하면 즐겨찾기 해제되고 즐겨찾기 메뉴 내 리스트에서도 사라진다.

설정 메뉴는 클릭하면 프로필 사진 변경 내 닉네임이 나오고 아래 닉네임 변경 버튼이 있고 비밀번호 변경, 이메일 변경, 알림 설정 이렇게 3개의 영역이 있어
닉네임 영역의 변경 버튼을 클릭하면 나의 닉네임이 입력된 채 수정할 수 있도록 입력필드가 활성화되면 좋겠고 닉네임 변경 버튼은 확인 버튼으로 변경돼어야해

그리고 비밀번호 변경 영역을 클릭하면 변경 페이지로 이동되고, 현재 비밀번호 확인란과 새 비밀번호 입력 필드 그리고 새 비밀번호 확인 입력 필드 3개 필드와 변경 버튼이 보여져

현재 비밀번호를 입력하고 새 비밀번호 입력과 입력 확인란까지 다 입력하고나서 변경 버튼을 클릭하면 변경되는 구조야
근데 현재 비밀번호가 다르거나 새 비밀번호 입력란과 입력 확인란이 다르거나 비밀번호 규칙에 맞지 않은 경우 변경 버튼 클릭 시 문제 있는 입력필드 하단에 빨간 텍스트로 안내 문구 노출

다음으로 이메일 영역에는 현재 등록한 이메일이 보여지며 변경 버튼 클릭 시 닉네임 변경과 동일하게 현재 이메일이 수정 가능한 활성화되고 변경
이메일로 우리는 사용자 인증을 하기때문에 이메일 변경 버튼은 인증하기 버튼으로 변경되고 버튼을 클릭하면 이메일 필드는 비활성화되고 입력된 이메일로 인증번호 숫자 6자리가 발송돼 (이건 회원가입 시 동일한 인증 방식)
그리고 인증번호를 입력할 수 있는 필드가 추가로 보여지고 확인 버튼도 함께 보여져
인증번호가 맞다면 이메일 필드 아래에 변경되었습니다. 라는 마이크로텍스트가 보여지고 인증번호가 틀리면 인증번호를 다시 확인해 주세요. 라는 문구가 보여져

알림 설정 같은 경우는 알림 ON/OFF 토글 스위치가 있어
디바이스 설정 메뉴로 이동해서 알림을 키도록 하는건 나중에 하자

공지사항 메뉴는 게시판 형식으로 공지사항 제목이 보여지고 년도와 월일이 있어
클릭하면 공지사항의 상세 내용을 확인할 수 있지

마지막으로 고객센터 메뉴는 메뉴 UI만 만들어놓고 클릭 시 어떻게 보여줄지는 좀 만 생각하자

로그아웃 버튼 클릭하면 로그아웃되고 브타 피드 화면으로 이동돼
