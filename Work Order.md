## 작업 ID
SCR-001 (로그인 화면)

## 작업 목표 (한 줄)
이번 작업의 목표는 프로토타입 디자인과 동일한 로그인 화면 UI 제작이다.

---

## 이번 작업에서 하는 것 (Do)
- [ ] UI 구현: @bmta_integrated_prototype.html 의 로그인 섹션 디자인을 1:1로 정밀하게 구현해줘.
- [ ] 테마 적용: 모든 스타일(색상, 폰트, 간격)은 @app_theme.dart 에 정의된 값을 사용하고, 수치적 간격은 8px 배수 시스템을 엄격히 준수해.
- [ ] 구조화: lib/views/login/login_view.dart 에 UI를 작성하되, 상태 관리는 Riverpod를 사용할 준비를 해줘. (일반 상태는 일단 UI 내에 두어도 좋지만, 구조는 깔끔하게 분리해줘.)
- [ ] 아이콘: 아이콘은 프로젝트 설정에 맞게 Lucide Icons를 사용해.

---

## 이번 작업에서 하지 않는 것 (Do NOT)
- ❌ 실제 Firebase Auth 로직 구현 금지 : 로그인 버튼을 눌렀을 때 실제로 Firebase와 통신하는 인증 로직은 작성하지 마. 현재는 버튼 클릭 시 콘솔에 로그만 찍거나 다음 화면으로 넘어가게만 해줘.(UI 집중을 위해)
- ❌ 외부 이미지 에셋 추가 금지 : 프로젝트 폴더에 없는 별도의 .png나 .svg 이미지 파일을 임의로 사용하지 마. 모든 그래픽 요소는 Flutter 코드로 그리거나 Lucide Icons로 대체해.(경로 에러 방지용)
- ❌ 글로벌 상태 관리(Provider) 완성 금지 : 로그인 상태를 관리하는 auth_provider.dart 같은 전역 상태 클래스를 지금 만들지 마. 이번엔 오직 login_view.dart 내에서의 UI 상태(입력값 확인 등)만 다뤄.
- ❌ 다른 화면(메인 피드 등)으로의 라우팅 연결 금지 : 로그인 성공 후 메인 화면으로 이동하는 정식 내비게이션 로직은 넣지 마. 일단 화면 자체의 완성도에만 집중해.
